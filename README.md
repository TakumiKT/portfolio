\## サービス概要

登録販売者向けの業務メモアプリ 



\## このサービスへの思い・作りたい理由

ドラッグストアで副店長として働く中、多くのOTC医薬品(一般用医薬品)の相談をお客様から受けました。

中には、登録販売者資格の学習でも会社研修でも学ばなかった相談内容も多く有りました。

上長に聞いたり、自分で調べる等していく中で、これを手軽にPC上で纏めて後から振り返る事が出来たらなと感じていました。

その経験から登録販売者向けに医薬品情報・接客事例・注意点を簡単に記録できるアプリを作成したいと考えに至りました。



\## ユーザー層について

本アプリは、登録販売者を対象者にしています。

ユーザーが日々の接客判断や注意点を個人で記録し、後から振り返ることで学習や判断精度の向上に繋げるための

個人利用向け業務メモアプリです。



\## サービスの利用イメージ

登録販売者が日々の業務の中での疑問点やナレッジを記録、振り返りをするときに利用する事をイメージしています。

\- 接客時に判断が難しかったケースを、業務後に振り返りたいとき

\- 同じような症状への対応を、過去の記録から確認したいとき

\- 自分の接客判断の傾向や弱点を整理したいとき



\## ユーザーの獲得について

本アプリは特定店舗への導入を前提とせず、登録販売者個人での利用を想定しています。

\- 現役の登録販売者

\- 転職・復職に向けて接客判断を整理したい方



\## サービスの差別化ポイント・推しポイント

* Microsoft Teams
* Notion

    \* これら2つのアプリは幅広い用途に対応できる一方で、

特定の情報を振り返ろうとした際に、見つける事に手間取る等の課題があります。



本アプリの特徴は、登録販売者が接客時に行っている判断プロセスを、

あらかじめ決められた観点で分解して記録できる点にあります。



一般的なノートアプリでは、

「接客内容を自由に書く」ことはできても、

どの観点で振り返ればよいかは利用者に委ねられています。

本アプリでは、接客メモを以下のような構造化された項目で記録します。



\- 相談された症状・状況

\- 接客時に確認したポイント

\- 自分が行った判断・対応内容

\- 注意した点や迷った点

\- 振り返って感じた課題や改善点



これらにより、単なる記録ではなく、

「なぜその判断をしたのか」「どこで迷ったのか」を

後から振り返りやすい形で知識として蓄積することができます。

登録販売者が自身の接客判断を言語化し、経験を学習につなげることに特化している点が、

既存の汎用ノートアプリとの大きな違いです。





\## 機能候補

* MVPリリース時に作っていたいもの

1. ユーザー登録、ログイン機能
2. 接客メモのCRUD
3. タグ機能(ユーザーによって作成)
4. 一覧、検索機能
5. 注意書き、免責事項（学習用途であることを明示する注意書き）



* 本リリースまでに作っていたいもの

1. 接客メモの振り返りレポート
2. より詳細な検索機能
3. CSVエクスポート

\## 検証・フィードバックについて

本アプリは特定店舗への導入を前提とせず、

登録販売者個人での利用を想定しています。

検証については、現役登録販売者や学習者へのヒアリング、

およびデモ環境での簡易ユーザーテストを通じて、

操作性や入力項目の妥当性についてフィードバックを得る計画です。
